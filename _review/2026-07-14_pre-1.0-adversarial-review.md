# Divine Hands — pre-1.0 adversarial review (2026-07-14)

Whole-mod review at HEAD `6c735e8`, Version 0.1.0. 6 dimension finders → triage → 2 adversarial
skeptics per finding (all Sonnet). **12 findings, all CONFIRMED by both skeptics** (0 disputed, 0
rejected). #1 and #5 independently re-verified against the decompile by hand.

Legend: 🔴 must-fix (functional/data bug) · 🟠 should-fix · ⚪ polish (optional for 1.0)

## 🔴 #1 — CRITICAL — autosave breaks lake persistence  `LakeStamp.cs:331`
`WriteSidecarForSave` derives the sidecar folder from `SaveManager.activeSaveFileName`, but FF's
`SaveInternal` **nulls that field for "AutoSave 1"/"AutoSave 2"** inside its own `finally` (via
`MoveSavMapFiles`, decompile 182852-854) **before** the Harmony postfix runs. So on any autosave the
sidecar is written **unfoldered** (`Save/AutoSave 1.divinehands-lakes.csv`), while the load path reads
the **foldered** key (`Save/<Settlement>/AutoSave 1…`, from `activeSaveFileName` set at 94628/103811).
Read misses → 0 lakes restored → dry basin remains, water gone. **This is the "lake didn't persist"
symptom.** Fires on every autosave-then-Continue with Persist Lakes on (default).
**Fix:** capture the foldered save key in a *prefix* (before FF nulls it) or derive the settlement
folder from the live game, not from the post-mutation `activeSaveFileName`. Must match the read key.

## 🔴 #5 — HIGH — fish stocking is 100% dead  `LakeStamp.cs:537-538`
`_waterAreaInfoType.GetField("id")` / `GetField("area")` — but `WaterAreaInfo.id`/`.area` are get-only
**auto-properties** (decompile 483387/483389), not fields. Both return null → `ok=false` (line 549) →
logs "fish stocking disabled" and the Stock Fish option never does anything. **Verified by hand.**
**Fix:** `GetProperty("id")` / `GetProperty("area")` (get-only, so read via `.GetGetMethod()` or
`GetValue`). One-line class of fix.

## 🔴 #2 — HIGH — DH stone pits balloon 8m→20m on reload  `CursorSpawners.cs:113`
`PitRadius = 8f` feeds `CreateStoneSite(Vector2,…)` (no floor), but `StoneSite.Load` **unconditionally**
does `radius = Mathf.Max(20f, radius)` (decompile 160048). On first save+reload the stone pit's collider
+ ground decal jump from ~16m to ~40m diameter and can overlap whatever was placed nearby. Stone-only
(Clay/Sand `.Load` have no floor). Reachable from Spawner *and* Mountain-brush deposit pass.
**Fix:** create stone sites at radius ≥20 (match FF's floor so what you place is what persists); leave
clay/sand at 8.

## 🔴 #4 — HIGH — persisted god-powers bleed into unrelated saves  `Plugin.cs:104-109`
`PersistRevealActive/PersistBuildAnywhereActive/PersistGodViewActive` are a **single global** pref (one
`CreateCategory("DivineHands")`, no per-save scoping) and are **never cleared on scene-exit**. Leave
Reveal Map on when you exit to menu → it re-activates on the **next map loaded, even a different colony**,
clearing that save's fog before you touch anything; save before noticing and full-explored is baked in.
Same for Build Anywhere / God View. Contradicts the modules' own "NOT a saved pref, reset every load"
doc comments.
**Fix (design choice):** per-save scoping (like the lake sidecar — key the flag to the save), **or**
the simpler "clear Persist* on scene-exit" (loses cross-reload persistence unless reload is same-session).
→ needs your call on which behavior you want.

## 🟠 #3 — HIGH — mountain brush edge seam with Ruggedness>0  `MountainBrush.cs:152`
`shape = Clamp01(dome + ridge*rug*(1-dome))` never attenuates the ridge term near the footprint edge —
Pangu gates it with a peak-proximity smoothstep (`num30`, Pangu 6366-67) that DH's port silently dropped
(not on the documented drop-list). At default Ruggedness 55% a ridge crest near the rim can raise the
last brushed cell several metres while the adjacent untouched cell stays flat → near-vertical seam
instead of the taper Edge Softness promises.
**Fix:** port Pangu's `num30` edge/peak-proximity gate onto the ridge term.

## 🟠 #6 — MEDIUM — SpawnPit deep-toggles the wrong site on failed placement  `CursorSpawners.cs:1193-1205`
The `bool` return of `CreateClay/Sand/StoneSite` is discarded; when Deep is on, code grabs
`list[list.Count-1]` regardless. If the create silently failed (off-grid, `pathingGridBounds` reject at
159784), it flips a **pre-existing unrelated** site infinite.
**Fix:** check the bool return, or snapshot `list.Count` before/after and only touch the new site.

## 🟠 #7 — MEDIUM — ForestBrush rect unclamped, edge trees silently dropped  `ForestBrush.cs:83-84`
Only brush footprint in the mod not clamped to world extents. Near a map edge it feeds out-of-range
worldX/Z into `AddGrowingTree`, whose tree-grid index has no bounds check → throws → caught & dropped.
Trees near the edge just don't plant.
**Fix:** clamp the rect like MountainBrush.cs:82-85 / FertilityBrush.cs:100-103 do.

## 🟠 #8 — MEDIUM — brush preview outline larger than actual edit near edge  `TerrainElevation.cs:481`
`TryGetGridGeometry` (preview) isn't clamped though every edit path is, so the cursor outline near a map
edge covers more than what gets edited (cosmetic/misleading, all brushes).
**Fix:** clamp the preview geometry, or route preview through the already-clamped `TryGetBrushRect`.

## ⚪ #9 — MEDIUM — per-frame reflection re-resolve  `TerrainElevation.cs:577,645`
`TryGetCursorWorldPoint`/`ResolveTerrain` tail re-`GetMethod`/`GetProperty` by name **every frame** any
brush is armed, unlike the cached handles beside them.
**Fix:** cache in static fields like `_terrainField` etc.

## ⚪ #10 — MEDIUM — non-atomic sidecar write  `LakeSidecar.cs:90`
`File.WriteAllText` straight over the live path — crash mid-write leaves a truncated/empty sidecar (FF
itself uses temp+rename `SaveInProgress.sav`). Read wraps the whole loop in one try/catch so one bad line
drops all lakes.
**Fix:** write temp then `File.Replace`/rename; parse per-line defensively.

## ⚪ #11 — LOW — mountain texture paints box, not circle  `MountainBrush.cs:278`
`PaintBiomeAndTexture` fills the full bounding box even when Shape=Circle (height sculpt already respects
the circle). Rocky texture spills to the corners.
**Fix:** pass `circle` through and gate the paint loop like `EdgeFactor`/`TryPickPoint` do.

## ⚪ #12 — LOW — hotkey spec re-parsed every frame  `Hotkey.cs:49`
`Pressed()/Held()` `Split('+')` + parse the spec string every call (7 hotkeys/frame), before even the
key-state check. Tiny, but trivially cacheable.
**Fix:** cache parsed spec keyed on the string.
